  V2 SCHEMA — 17 top-level fields per record:
    id                       string (lowercase + underscores)
    biome                    string (exact biome name)
    variant                  string
    width                    "3m" | "6m" | "12m" | "20m"
    assumedBiomassHeight     string
    bespokePlantingRegime    string
    agronomicWindProfile     string
    carbonSequestration      { yr_1_5 … yr_46_50 } — 10 × 5yr period floats
    sepaMetrics              { riparian, crossSlope, downSlope } × 3 scores 0–100
    crewValuations           totals + airFiltration + catchmentHydrology
                             + waterPurification sub-objects
    radarMetrics             4 global scores + placement-specific hydro/water
    farmImpacts              4 farm types × 3 value fields
    economicAssessment  *** v2 new *** — 3 placements × 5 avoided-cost components
                             + windbreakNS + windbreakEW
    advancedEcosystemServices *** v2 new *** — pestRegulation, thermalRegulation
                             (by farm type) + fioTrappingEfficiency
    registry            *** v2 new *** — 11 fields incl. sserGrossUnitsPerKm
    seq50yrTotal        *** v2 new *** — pre-computed 50yr carbon sum

DB.JS — EXPORTS
  openDB()            Opens IndexedDB "FifeShieldBelt" v2. On upgrade/first load:
                      drops old store, creates "interventions" store (keyPath: id),
                      creates by_biome index, fetches JSON, bulk-inserts all 120
                      records in a single onupgradeneeded transaction.
                      Logs: "FifeShieldBelt DB v2 seeded: 120 records" on seed
                            "FifeShieldBelt DB: loaded existing v2 store" on reload

  dbReady             Exported Promise — app.js awaits this before first query

  getAll()            Returns all 120 records
  getByBiome(name)    Uses by_biome index; returns 20 records for that biome
  getById(id)         Single record lookup by string id
  getBiomes()         Reads unique biome keys from index cursor (sorted)
  validateStore()     Dev helper — checks all v2 schema rules, logs console.table
                      of any violations. Rules checked:
                        - Exactly 120 records, 20 per biome
                        - id format (lowercase + underscores)
                        - width in allowed set
                        - carbonSequestration: all 10 periods ≥ 0
                        - seq50yrTotal cross-check (within 0.01 tolerance)
                        - sepaMetrics: all scores 0–100
                        - crewValuations totals ≥ 0 + 3 sub-objects present
                        - farmImpacts: all 4 farm types, all 3 fields each
                        - economicAssessment present
                        - advancedEcosystemServices.fioTrappingEfficiency 0–100
                        - registry: all 11 keys present
                        - sserGrossUnitsPerKm in 9-tier allowed set

  BIOMES              Exported const object — 6 biome string values
  FARM_TYPES          Exported array — ['Cereals','General Cropping','Dairy','LFA Grazing']
  WIDTHS              Exported array — ['3m','6m','12m','20m']
  SSER_TIERS          Exported array — [3.96, 5.28, 7.92, 10.56, 11.88,
                                         15.84, 19.8, 23.76, 39.6]

  DB_VERSION = 2      Forces onupgradeneeded / re-seed if browser cached v1

CALC.JS — EXPORTS
  calculate(inputs, record)
    Master function. Returns full results object (35+ fields).
    Inputs: { biome, variantId, farmType, placement, lengthM,
              creditPrice, windbreakOrient }

  totalSeq50yr(record)
    Reads record.seq50yrTotal directly (no re-summation of periods).

  cumulativeSeqTrajectory(carbonSequestration)
    Returns 10-element array of running totals across 5-year periods.
    Used by the trajectory line chart.

  scaleToLength(perKmValue, lengthM)
    perKmValue × (lengthM / 1000)

  annualCarbonIncome(record, lengthM, creditPrice)
    Formula: (seq50yrTotal / 50) × (widthM × lengthM / 10000) × creditPrice

  netIncomeForegone(record, farmType, lengthM)
    record.farmImpacts[farmType].netIncomeForegone scaled to length.

  pollinationValue(record, farmType, lengthM)
    record.farmImpacts[farmType].pollinationValue scaled to length.

  crewValue(record, placement, lengthM)
    Top-level placement total (riparianTotal / crossSlopeTotal /
    downSlopeTotal) scaled to length.

  crewBreakdown(record, placement, lengthM)   *** v2 new ***
    Disaggregated: airFiltration + catchmentHydrology + waterPurification
    each read from crewValuations[subKey][placement] and scaled.

  pestRegulationValue(record, farmType, lengthM)   *** v2 new ***
    record.advancedEcosystemServices.pestRegulation[farmType] scaled.

  thermalRegulationValue(record, farmType, lengthM)   *** v2 new ***
    record.advancedEcosystemServices.thermalRegulation[farmType] scaled.

  windbreakValue(record, orientation, lengthM)   *** v2 new ***
    Sum of yieldBump + waterRetention + nutrientPump from
    economicAssessment.windbreakNS or .windbreakEW, scaled.

  windbreakBreakdown(record, orientation, lengthM)   *** v2 new ***
    Per-component object: { yieldBump, waterRetention, nutrientPump }

  avoidedCosts(record, placement, lengthM)   *** v2 new ***
    Sum of 5 components from economicAssessment[placement]:
    floodControl + avoidedErosion + avoidedCompaction +
    avoidedDownErosion + avoidedFuelCosts, scaled.

  avoidedCostsBreakdown(record, placement, lengthM)   *** v2 new ***
    Per-component object for all 5 avoided-cost fields.

  sserUnits(record, lengthM)   *** v2 new ***
    record.registry.sserGrossUnitsPerKm × (lengthM / 1000)

  parseWidth(widthStr)
    '6m' → 6

  fmt(n, dp)     toLocaleString('en-GB') with fixed decimal places
  fmtGBP(n)      '£' + rounded integer with thousand separators
  fmtCarbon(n)   fmt(n, 1) + ' tCO₂e'
  fmtSSER(n)     fmt(n, 2) + ' units'

  annualNetBenefit formula:
    carbonIncome + pollinationValue + crewValueTotal
    + pestRegulationValue + thermalRegulationValue
    + windbreakValue + avoidedCosts
    − netIncomeForegone

  All numeric fields are guarded by safeNum() — engine never produces NaN
  regardless of missing or null fields in sparse records.
